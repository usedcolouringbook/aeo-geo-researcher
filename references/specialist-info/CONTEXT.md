# References — Entry Point

Shared knowledge library for all specialists. Navigate from here — do not link directly to individual files in persistent routing documents.

Full index with load guidance: `references/specialist-info/INDEX.md`

## When to load what

Each specialist's `rules.md` contains a Reference Library table listing the specific files relevant to its investigation. Load only those files for the current session, not the full library.

| Reference file | Load when... |
|---|---|
| `aeo-geo-signals.md` | Investigating RAG mechanics, crawl signals, schema, chunking, llms.txt, or SoM |
| `aeo-audit-checklist.md` | Running a structured technical or content audit |
| `geo-content-framework.md` | Investigating content structure, Answer Capsule format, or citation readiness |
| `platform-behaviors.md` | Investigating per-platform citation mechanics (ChatGPT, Perplexity, Google, Claude, Copilot) |
| `rag-taxonomy.md` | Client involves RAG systems or agent pipelines with retrieval components |
| `agentic-patterns.md` | Investigating MCP, agent pipelines, or agentic marketing strategy |
| `source-credibility.md` | Always — loaded by all specialists; defines the two-axis finding model |
| `glossary.md` | Domain terminology lookup |
| `../manifest.yaml` | Orchestrator only — packet schema for inter-specialist handoffs |
| `../report-template.html` | Claude Projects sessions — HTML export on user request |
